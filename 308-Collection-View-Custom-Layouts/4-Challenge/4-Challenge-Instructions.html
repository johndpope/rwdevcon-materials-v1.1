Challenge 1

You need to figure out how you properly calculate frame, and then replace the // TODO with the necessary code.

Things to think about:

  • The tracks should run down the left-hand side of the schedule.
  • Just like the sessions, their position should take the height of the hour header views into account.
  • You already have the width (trackHeaderWidth) and height (heightPerTrack) of each track.

Finished? Okay, let’s continue …

The next step is to update prepareLayout() so it calculates and caches the frames of the track headers. Add the following at the bottom of the method:

if trackHeaderFrames.isEmpty {  
  for track in 0..<dataSource.numberOfTracksInSchedule {  
    let indexPath = NSIndexPath(forItem: track, inSection: 0)  
    let rect = frameForTrackHeaderViewAtIndexPath(indexPath)  
    let frame = Frame(rect: rect, indexPath: indexPath)  
    trackHeaderFrames.append(frame)  
  }
}

This code is almost identical to that which you wrote for the hour header frames; you iterate over the tracks in the schedule, generate an index path, calculate the frame, and cache the two by wrapping them in the Frame struct and appending that to the trackHeaderFrames array.

Next, you need to update layoutAttributesForElementsInRects(_:) so the layout attributes for any track headers intersecting with the given rect are created and returned to the collection view. Add the following to the bottom of the method, just before the return statement:

for frame in trackHeaderFrames {  
  if CGRectIntersectsRect(frame.rect, rect) {  
    let attributes = UICollectionViewLayoutAttributes(forSupplementaryViewOfKind: "TrackHeader",  
      withIndexPath: frame.indexPath)  
    attributes.frame = frame.rect  
    attributes.zIndex = -1  
    layoutAttributes.append(attributes)  
  }
}

Again, this is almost identical to the code you wrote to generate the layout attributes for the hour headers, except this time you use TrackHeader as the kind so the two different kinds of supplementary view can be told apart.

There are three remaining changes you need to make to ScheduleLayout.swift in order for the track headers to be displayed properly. Since the track headers will be displayed down the left-hand side of the collection view, their width needs to be taken into account when laying out both the session cells and the hour header views.

Challenge 2

You need to update the following three methods in order to get the track headers to display properly:

  1. collectionViewContentSize()

Finished? Okay, let’s continue …

Open ScheduleViewController.swift and, in viewDidLoad(), register the headerNib with the collection view a second time, but this time passing TrackHeader as the kind.

Finally, remove the contents of the header configuration block, replacing them with the following:

if kind == "HourHeader" {  
  header.titleLabel.text = dataSource.titleForHourHeaderViewAtIndexPath(indexPath)  
} else if kind == "TrackHeader" {  
  header.titleLabel.text = dataSource.titleForTrackHeaderViewAtIndexPath(indexPath)  
  if let verticalSeparatorView = header.verticalSeparatorView {  
    verticalSeparatorView.hidden = true  
  }
}

This sets the title of the track header view using a convenience method on the data source, and since you’re reusing the same nib as the hour header, you also hide the vertical separator.
