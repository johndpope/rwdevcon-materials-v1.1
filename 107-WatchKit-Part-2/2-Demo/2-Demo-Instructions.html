107: WatchKit II, Part 2: Demo Instructions

In this demo, you will add a context menu and Glance to the RWDevCon WatchKit App. The steps here will be explained in the demo, but here are the raw steps in case you miss a step or get stuck.

Please make sure you use the starter project provided, rather than continuing with the project you created in Greg’s session. This is because there might be subtle differences, and not everyone will have completed the challenge

Context Menus

1. Context Menus in Interface Builder

Open Interface.storyboard, which can be found in the RWDevCon WatchKit App group, and drag a Menu from the Object Library onto DetailsInterfaceController in the storyboard canvas.

Expand DetailsInterfaceController Scene in the Document Outline to locate the new menu. Expand it to find the single menu item, and select it. In the Attributes Inspector change the Title to Favorite and the Image to Add.

With the RWDevCon WatchKit App scheme selected, build and run. Tap the first session, and then long-press to invoke the context menu. Tap anywhere to dismiss it. Stop the app.

2. Context Menus in Code

Delete the Menu you just added to DetailsInterfaceController.

Open DetailsInterfaceController.swift, which can be found in the RWDevCon WatchKit Extension group.

Add the following code to the top of awakeWithContext(_:), just below the call to super:

addMenuItemWithItemIcon(.Add, title: "Favorite", action: "toggleFavorite")

Build and run. Tap the first session, and then long-press to invoke the context menu. Tap anywhere to dismiss it. Stop the app.

3. Dynamic Context Menus

Delete the code you just added to awakeWithContext(_:).

Add the following property to the top of DetailsInterfaceController, just below the outlets:

private var session: Session?

Add the following method to the bottom of the class:

func configureContextMenu() {  
  clearAllMenuItems()  
  if let session = session {  
    let favorite = session.isFavorite  
    let icon: WKMenuItemIcon = favorite ? .Decline : .Add  
    let title = favorite ? "Unfavorite" : "Favorite"  
    addMenuItemWithItemIcon(icon, title: title, action: "toggleFavorite")  
  }
}

Just below that, add the implementation of toggleFavorite:

func toggleFavorite() {  
  if let session = session {  
    session.isFavorite = !session.isFavorite  
    configureContextMenu()  
  }
}

The final step is to set the session property in awakeFromContext(_:) and create the initial context menu. Add the following to the very bottom of the if let session = context... if block in awakeFromContext(_:):

self.session = session  
configureContextMenu()

You set up the initial context menu in the first if block because this interface controller is used for both session and speaker details, but you only want the menu available when the user is viewing a session.

Build and run. This time tap the Keynote session, and then long-press to invoke the context menu. Tap the icon to favorite the session and dismiss the menu. Long-press again to invoke the menu a second time; this time the icon and title will be different, and you’ll be able to unfavorite the session.

Bug Alert: There is currently an issue in Beta 4 relating to Context Menus being presented from modal interface controllers. The first time the modal interface controller is presented everything will work as expected, but presenting the modal interface controller a second time and invoking the Context Menu will cause the app to hang, and eventually crash. I had a Radar closed as a duplicate, so Apple are obviously aware of the issue.

Please make sure you have favorited the “Keynote” session before moving onto the next part.

Glances

1. Creating the Glance Interface

Open Interface.storyboard from the RWDevCon WatchKit App group and drag a Glance Interface Controller from the Object Library onto the storyboard canvas.

Select the top layout group and in the Attributes Inspector change the Insets to Custom, with a value of 4 for both Left and Right.

Drag an Image from the Object Library into the top layout group of the Glance Interface Controller. In the Attributes Inspector change both the Width and Height to Fixed, and change their values to 30. Change the Vertical position to Center. Finally, change the Image to RW_logo.

Select the bottom layout group of the Glance Interface Controller and in the Attributes Inspector change it’s Layout to Vertical, Insets to Custom with a value of 4 for both Left and Right, and finally set Spacing to 2. Then, from the Object Library, drag a Label into that layout group, followed by a Group, and then another Label.

Select the new layout group and in the Attributes Inspector change Spacing to 6.

Drag an Image and a Label into the new layout group. Select the image, and in the Attributes Inspector change both the Width and Height to Fixed and set their values to 22. Set the Vertical position to Center.

Next, select the top Label of the three. In the Attributes Inspector change Text to Session, and Font to System with a size of 34.0.

Then, select the middle Label. In the Attributes Inspector change Text to Time, Font to System with a size of 26.0, and Vertical position to Center.

Finally, select the bottom Label. In the Attributes Inspector change Text to Room, Text Color to #006D37, Font to System Bold with a size of 15.0, and Min Scale to 0.5.

2. Running the Glance

Before you can run the Glance you need to create a custom scheme.

From the Xcode menu select Product\Scheme\Manage Schemes to open the Scheme Editor.

Select the RWDevCon WatchKit App scheme and then click the gear icon. From the popup menu choose Duplicate. In the next dialog name the scheme RWDevCon WatchKit Glance.

Select the Run option on the left and then change Watch Interface to Glance.

Click Close to save the changes.

Next, select RWDevCon WatchKit Glance from the scheme menu and build and run. You should see your Glance appear in the simulator. You can verify everything is running as expected by tapping the Glance, which should then launch the Watch App.

3. Populating the Glance

Right-click on the RWDevCon WatchKit Extension group and choose New File…, then choose iOS\Source\Swift File and click Next. Name the new file UpcomingGlanceController.swift and click Create.

Open UpcomingGlanceController.swift and replace the existing import statement with the following:

import WatchKit

And then add the following class definition:

class UpcomingGlanceController: WKInterfaceController {

}

Next, add the following outlets to the top of the class for the interface elements you added earlier:

@IBOutlet weak var sessionLabel: WKInterfaceLabel!  
@IBOutlet weak var timeImage: WKInterfaceImage!  
@IBOutlet weak var timeLabel: WKInterfaceLabel!  
@IBOutlet weak var roomLabel: WKInterfaceLabel!

Open Interface.storyboard and select Glance Interface Controller, either in the storyboard canvas or the Document Outline. Then, in the Identity Inspector, set the Class to UpcomingGlanceController.

Next, right-click on Glance Interface Controller in the Document Outline and from the popup menu connect each of the outlets to their corresponding interface element.

Open UpcomingGlanceController.swift and declare the following lazy property:

lazy var coreDataStack = CoreDataStack()

And then add the following:

override func awakeWithContext(context: AnyObject?) {  
  super.awakeWithContext(context)

  if let session = Session.nextFavoriteSession(coreDataStack.context) {  
    if let clockImage = Clock.imageForSession(session) {  
      timeImage.setImage(clockImage)  
    } else {  
      timeImage.setHidden(true)  
    }
    sessionLabel.setText(session.title)  
    timeLabel.setText(session.startTimeShortString)  
    roomLabel.setText(session.room.name)  
  }
}

Here you request the next favorite session when the Glance is about to be shown, and if one is returned use it to populate the various interface elements.

With the RWDevCon WatchKit Glance scheme selected, build and run. You’ll now see the Glance displays the details of your next favorite session.
