107: WatchKit II, Part 3: Lab Instructions

At this point, you’ve added a Glance which displays the details of your next favorite session.

However, wouldn’t it be nice if, after tapping the Glance to launch the WatchKit App, the associated session details where shown automatically. It sure would! And you can do exactly that using Handoff.

There are two steps to implementing Handoff support:

  1. The Glance needs to advertise the data it’s displaying
  2. The WatchKit App needs to use that data to show the appropriate interface controller at launch

And that’s what you’ll be working on in this lab!

Note: This Lab is intended to be relatively easy as I want to give everyone a shot at completing both the Lab and the Challenge, thereby putting the finishing touches to the WatchKit app.

Glance, meet WatchKit App

You’ll start by getting the Glance to talk to the WatchKit App.

Open UpcomingGlanceController.swift from the RWDevCon WatchKit Extension group and add the following to the bottom of the if block inside awakeWithContext(_:), just below where you’re setting the roomLabel text:

updateUserActivity("com.razeware.RWDevCon.Glance", userInfo: [“foo”: “bar”])

Calling this method tells WatchKit to call the main interface controller’s handleUserActivity(_:) method at launch. You pass a unique string to identify the Handoff activity, and a dictionary providing contextual information which the WatchKit App can then use to configure its interface.

Next, open InterfaceController.swift; this is the WatchKit App’s main interface controller. Add the following method just below awakeWithContext(_:):

override func handleUserActivity(userInfo: [NSObject : AnyObject]!) {  
  println("Hello, WatchKit App!")  
}

This method is called when the WatchKit App is launched, so long as the Glance has updated it’s activity, which you did in the previous step. Here you’re simply printing a message to the console.

With the RWDevCon WatchKit Glance scheme selected, build and run. Tap the Glance, and once the WatchKit App has finished launching you should see the following in the console:

Hello, WatchKit App!

Fantastic! You’ve actually completed the hardest part: getting the Glance and WatchKit App talking.

Identify Yourself!

So far the contextual information you’re passing from the Glance to the WatchKit App is anything but.

You need to pass something meaningful to the WatchKit App which it can then use to update it’s interface accordingly. As each key and value of the userInfo dictionary must be of the following types:

  • NSArray
  • NSData
  • NSDate
  • NSDictionary
  • NSNull
  • NSNumber
  • NSSet
  • NSString
  • NSURL

…passing the session object itself is out of the question. Instead you’ll pass the identifier property of the session, which the WatchKit App can then use to fetch the corresponding session from the Core Data store.

Open UpcomingGlanceController.swift and replace the existing updateUserActivity(_:userInfo:) call with the following:

updateUserActivity("com.razeware.RWDevCon.Glance", userInfo: ["identifier": session.identifier])

Here you’ve updated the userInfo dictionary so it now contains a key called identifier which has a value of the session’s identifier.

Next, open InterfaceController.swift and replace the contents of handleUserActivity(_:) with the following:

if let identifier = userInfo["identifier"] as? String {  
  println(identifier)  
}

Here you simply extract the identifier from the userInfo dictionary and print it to the console.

With the RWDevCon WatchKit Glance scheme still selected, build and run. Tap the Glance, and once the WatchKit App has finished launching you should see the session identifier in the console:

keynote1

Note: As long as you favorited the Keynote session then the identifier you’re seeing in the console should match the above. If you favorited a different session then it’ll obviously be different.

Now you have the identifier you can use it to present the session details.

Open the session, please, HAL.

The final step is to fetch the corresponding session from the Core Data store and present its details to the user.

Open InterfaceController.swift and replace the println() call in handleUserActivity(_:) with the following:

// 1  
if let session = Session.sessionByIdentifier(identifier, context: coreDataStack.context) {  
  // 2  
  let presenters = session.presenters.array  
  // 3  
  presentControllerWithNames(Array<String>(count: presenters.count + 1,  
    repeatedValue: "DetailsInterfaceController"), contexts: [session] + presenters)  
}

Here what’s happening:

  1. Fetch the session using the received identifier
  2. Extract the array of presenters
  3. Present a page-based set of interface controllers; one for the session itself, and one for each presenter

With the RWDevCon WatchKit Glance scheme selected, build and run. Tap the Glance and, once the WatchKit App has finished launching, behold as the corresponding session and presenter details are displayed:
